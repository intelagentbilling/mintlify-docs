## Overview

The Records tab displays all your submitted claims, allowing you to track status, view details, and manage drafts.

## Accessing Records

Tap the **Records** tab (second icon) in the bottom navigation.

## Records List

Each record shows:
- Patient name
- Service date
- Billing codes
- Claimed amount
- Status badge

## Status Badges

Claims are color-coded by status:

| Status | Color | Meaning |
|--------|-------|---------|
| Draft | Blue | Saved, not submitted |
| Submitted | Gray | Sent to OHIP |
| Partial | Yellow | Partially paid |
| Successful | Green | Fully paid |
| Rejected | Red | Denied, needs attention |

## Filtering Records

### By Status

Tap the status filter at the top to show:
- All records
- Drafts only
- Rejected only
- Partial payments
- Submitted (pending)

### By Date

Tap the filter icon to set a date range:
1. Select start date
2. Select end date
3. Apply filter

### Search

Use the search bar to find records by:
- Patient name
- Health card number

## Viewing Claim Details

Tap on any record to see full details:
- Patient information
- Service details
- Billing codes and fees
- Diagnostic codes
- Attached documents
- Status history

## Editing Records

### Draft Claims

Drafts can be fully edited:
1. Tap the draft record
2. Modify any fields
3. Tap **Save** or **Submit**

### Submitted Claims

Limited editing for submitted claims:
- View details (read-only)
- Some rejected claims can be corrected and resubmitted

## Actions

### Edit

Open the claim for editing (drafts and some rejected claims).

### Delete

Remove a draft claim:
1. Tap the claim
2. Tap the trash icon
3. Confirm deletion

<Warning>
Submitted claims cannot be deleted from mobile. Contact support if needed.
</Warning>

### Resubmit

For rejected claims:
1. View the claim details
2. Review rejection reason
3. Make corrections
4. Tap **Resubmit**

## Pull to Refresh

Pull down on the records list to refresh and load the latest data from the server.

## Pagination

Scroll down to load more records. The list loads incrementally for performance.

## Offline Access

Recently viewed records are cached for offline access. Full functionality requires an internet connection.
